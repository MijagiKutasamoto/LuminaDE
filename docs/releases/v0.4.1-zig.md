# LuminaDE v0.4.1 ZIG (dev)

v0.4.1 skupia się na rozbudowie panelu i przejściu od bridge/fallback do natywnej ścieżki `layer-shell` jako domyślnej.

## Scope

- Pixel-pass kompozycji UI (panel/launcher/settings) w kierunku Aurora Glass.
- Start domykania ostatniego punktu Etapu 1.5: pełny natywny panel `layer-shell`.
- Wejście w Etap 2 przez integracje systemowe panelu.

## Zrobione na starcie v0.4.1

- Dodany strict native mode toggle:
  - `LUMINADE_NATIVE_PANEL_REQUIRED=1`
  - Gdy aktywny, panel nie schodzi do fallback TSV i wymaga działającej ścieżki native.
- Domknieta natywna sekwencja mapowania panelu layer-shell:
  - commit inicjalny -> `configure` event -> `ack_configure` -> finalny commit.
  - ten etap pozwala zaczac testy srodowiska na natywnej sciezce panelu.
- Panel dostal stale widgety systemowe (MVP Etapu 2):
  - `sys-net`, `sys-audio`, `sys-power`.
  - obsluga klikniec przez fallback komend systemowych (network/audio/lock).
- Panel dostal statusowe etykiety runtime:
  - `Net:...`, `Vol:...`, `Bat:...` (z fallback probe chain).
- Akcje systemowe zostaly wyniesione do `luminade-core` jako wspolny helper
  (wykorzystywany przez panel i settings).
- Settings dostaly akcje sieci (`Open Network`) pod wspolny helper systemowy.
- Dodany skrypt smoke testow:
  - `scripts/smoke-v041.sh --prepare-only`
  - `scripts/smoke-v041.sh --run`
- Native panel fallback state jest teraz per-output:
  - domyslnie: `.luminade/native-panel-state-<output>.tsv`
  - `LUMINADE_NATIVE_PANEL_STATE` wspiera placeholder `{output}`
- Runtime GUI metadata dla widgetow ma natywne wsparcie etykiet ikon:
  - format: `icon:<icon-name> <text>`
  - `luminade-ui` udostepnia `parseIconLabel` i `composeIconLabel`

## Kolejne kroki (shortlist)

- Lifecycle `wl_surface`/`zwlr_layer_surface_v1` utrzymywany stale (nie tylko smoke-create).
- Obsługa `configure`/`ack_configure` + commit loop pod panel native.
- Per-output natywny state panelu i routowanie eventów bez bridge-first (state per-output zrobiony, routing native-loop dalej do domkniecia).
- Integracja widgetów systemowych panelu: audio/sieć/zasilanie.
